# 0003 — Flujo comercial ecommerce

## Estado

Aceptada.

## Decisión

El ecommerce de Ofiponiente no será únicamente checkout directo.

Debe soportar varios caminos comerciales:

- Compra directa para productos simples.
- Solicitud de presupuesto para proyectos o pedidos complejos.
- Validación comercial cuando haya montaje, transporte, volumen o configuración.
- Anticipo del 50% en proyectos bajo pedido cuando corresponda.
- Pago restante antes de montaje o entrega.

## Contexto

Ofiponiente trabaja principalmente con proyectos, pedidos a fabricante, transporte y montaje.

## Consecuencias

- No imponer un único checkout a todos los productos.
- Las fichas deben poder convertir a compra, reserva o presupuesto.
- El CRM y el seguimiento comercial son parte central del sistema.

## Regla

El ecommerce debe apoyar al equipo comercial, no sustituirlo completamente.
