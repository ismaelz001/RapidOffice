# UX FRAMEWORK

> Framework oficial para analizar y diseñar la experiencia de usuario de Ofiponiente Digital.
>
> Este documento NO define diseño visual concreto.
> Define los principios de experiencia que deberá seguir cualquier web, configurador, catálogo o herramienta digital del proyecto.

# 1. Objetivo

Crear una experiencia digital que haga fácil comprar, configurar o solicitar una oficina.

El usuario no debe sentirse ante un catálogo infinito.

Debe sentirse guiado.

---

# 2. Principio central

El cliente no quiere navegar miles de productos.

Quiere resolver una necesidad.

Por tanto, la experiencia debe partir de preguntas como:

- ¿Qué espacio necesitas montar?
- ¿Cuántas personas trabajarán?
- ¿Qué presupuesto tienes?
- ¿Prefieres nuevo, outlet o reacondicionado?
- ¿Necesitas montaje?

---

# 3. Problema principal del sector

Muchas webs de mobiliario de oficina funcionan como catálogos.

Problemas habituales:

- demasiadas categorías
- demasiados productos
- poca orientación
- poca diferenciación
- poca claridad de precios
- poca visualización del resultado
- exceso de fricción para contactar

Ofiponiente debe evitar este patrón.

---

# 4. Experiencia objetivo

El usuario debe poder avanzar por tres caminos principales.

## Camino 1: Sé lo que quiero

Usuario que busca un producto concreto.

Debe encontrarlo rápido mediante:

- categorías claras
- buscador
- filtros
- fichas completas
- CTA directo

## Camino 2: Necesito montar una oficina

Usuario que no sabe qué productos elegir.

Debe ser guiado mediante:

- configurador
- packs
- ejemplos
- casos reales
- presupuesto orientativo

## Camino 3: Quiero hablar con un experto

Usuario que prefiere trato humano.

Debe tener acceso rápido a:

- llamada
- WhatsApp
- formulario
- solicitud de visita
- asesoramiento

---

# 5. Principios de navegación

La navegación debe ser simple.

Menú principal recomendado:

- Sillas
- Mesas
- Almacenamiento
- Recepción
- Outlet
- Proyectos
- Configura tu oficina

Evitar menús excesivamente técnicos.

El usuario debe entender cada sección sin conocer el sector.

---

# 6. Jerarquía visual

Cada página debe responder rápidamente:

1. Qué es esto.
2. Para quién es.
3. Qué problema resuelve.
4. Qué opciones tengo.
5. Qué hago ahora.

Si una página no responde estas cinco preguntas, está mal diseñada.

---

# 7. Fichas de producto

Una ficha de producto no debe ser solo una ficha técnica.

Debe ayudar a decidir.

Debe incluir:

- imágenes claras
- descripción útil
- medidas
- acabados
- fabricante
- tipo comercial
- plazo estimado
- opciones relacionadas
- alternativa nuevo / outlet / reacondicionado si aplica
- CTA claro

---

# 8. Configurador

El configurador debe ser sencillo.

No debe empezar como plano vacío.

Flujo recomendado:

Cuestionario

↓

Propuesta automática

↓

Visualización orientativa

↓

Contacto comercial

↓

Opcionalmente plano editable

El usuario no debe diseñar desde cero.

El sistema debe proponer primero.

---

# 9. Packs

Los packs son una herramienta UX y comercial.

Reducen decisión.

Ejemplos:

- Oficina para 2 personas
- Oficina para 5 personas
- Oficina para 10 personas
- Sala de reuniones
- Pack recepción
- Pack coworking
- Pack academia

Los packs deben mostrar precio orientativo y posibilidad de personalización.

---

# 10. Confianza

La confianza debe aparecer antes del precio.

Elementos recomendados:

- años de experiencia
- fabricantes
- showroom
- almacén
- logística
- montaje
- casos reales
- reseñas
- fotografías reales
- contacto visible

---

# 11. Fricción

Eliminar fricción innecesaria.

Evitar:

- formularios largos al inicio
- pedir demasiados datos pronto
- ocultar información básica
- obligar a llamar para todo
- navegación confusa
- CTAs ambiguos

---

# 12. CTAs

Cada página debe tener un CTA principal.

Ejemplos:

- Solicitar presupuesto
- Configurar mi oficina
- Consultar disponibilidad
- Pedir asesoramiento
- Reservar con 50%

No usar CTAs genéricos como "Enviar" si puede evitarse.

---

# 13. Mobile First

La experiencia móvil es prioritaria.

Todo deberá funcionar correctamente en móvil:

- menú
- filtros
- formularios
- configurador
- fichas
- contacto
- WhatsApp

Si algo no funciona en móvil, no está terminado.

---

# 14. Accesibilidad

La experiencia debe ser clara y usable.

Criterios básicos:

- buen contraste
- textos legibles
- botones grandes
- navegación con teclado
- etiquetas en formularios
- imágenes con alt
- estructura semántica

---

# 15. Rendimiento

La velocidad forma parte de la UX.

Priorizar:

- imágenes optimizadas
- carga rápida
- navegación fluida
- evitar scripts innecesarios
- evitar animaciones pesadas

Una web lenta reduce confianza y conversión.

---

# 16. Contenido visual

El contenido visual debe ayudar a imaginar el resultado.

Priorizar:

- oficinas reales
- antes / después
- montajes
- renders útiles
- imágenes de producto limpias
- comparativas

No usar imágenes decorativas sin función.

---

# 17. Personalización

La experiencia debe tender hacia la personalización.

Ejemplos:

- recomendaciones según tipo de empresa
- opciones según presupuesto
- alternativas nuevo / outlet / reacondicionado
- propuestas por número de empleados
- sugerencias por espacio

---

# 18. Errores a evitar

No construir:

- una web bonita pero difícil de usar
- un catálogo infinito sin guía
- un configurador demasiado complejo
- formularios interminables
- páginas sin CTA
- diseño visual que perjudique SEO
- animaciones que ralenticen

---

# 19. Métricas UX

Medir:

- tasa de rebote
- tiempo en página
- scroll
- clics en CTA
- uso del configurador
- formularios iniciados
- formularios completados
- conversión por dispositivo
- abandono en checkout o solicitud

---

# 20. Resultado esperado

La UX debe conseguir que el usuario piense:

"Entienden lo que necesito."

"Me están guiando."

"Parece una empresa seria."

"Puedo pedir presupuesto sin complicarme."

La experiencia debe convertir complejidad en claridad.
