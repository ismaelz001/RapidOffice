# SEO FRAMEWORK

> Framework oficial para cualquier auditoría SEO del proyecto Ofiponiente Digital.
>
> Objetivo:
> Analizar cualquier sitio web utilizando la misma metodología y criterios.
>
> Este documento NO contiene auditorías.
> Contiene únicamente la metodología.

---

# 1. Objetivo

Antes de proponer mejoras SEO, primero debemos comprender cómo está construida una web.

Nunca se propondrán cambios sin analizar previamente la arquitectura.

---

# 2. Orden obligatorio de análisis

Toda auditoría SEO seguirá exactamente este orden.

1. Negocio
2. Arquitectura
3. Indexación
4. Contenido
5. Intención de búsqueda
6. Autoridad
7. Experiencia de usuario
8. Conversión
9. Prioridades

No alterar este orden.

---

# 3. Negocio

Responder siempre:

- ¿Qué vende?
- ¿A quién vende?
- ¿Qué problema resuelve?
- ¿Cuál es su propuesta de valor?
- ¿Qué tipo de cliente busca?

Sin comprender el negocio no puede hacerse SEO correctamente.

---

# 4. Arquitectura

Analizar:

- Estructura de categorías.
- Profundidad de navegación.
- URLs.
- Breadcrumbs.
- Jerarquía.
- Enlazado interno.
- Landing Pages.
- Clusters.

Preguntas:

¿La arquitectura tiene sentido?

¿Es escalable?

¿Existe canibalización?

---

# 5. Indexación

Comprobar:

- Páginas indexadas.
- Robots.txt.
- Sitemap.
- Canonicals.
- Meta Robots.
- Duplicidades.

Herramientas:

- Google Search Console
- site:dominio.com
- Screaming Frog

---

# 6. Palabras clave

Analizar:

- Keywords principales.
- Keywords secundarias.
- Long Tail.
- Intención de búsqueda.

Clasificar por:

- Informacional.
- Comercial.
- Navegacional.
- Transaccional.

---

# 7. Contenido

Evaluar:

- Calidad.
- Profundidad.
- Originalidad.
- Actualización.
- Utilidad.

Preguntas:

¿Responde realmente la intención de búsqueda?

¿Aporta más valor que la competencia?

---

# 8. SEO On Page

Analizar:

- Title
- Meta Description
- H1
- H2
- H3
- URLs
- Alt
- Schema
- Open Graph

---

# 9. SEO Técnico

Analizar:

- Core Web Vitals
- Velocidad
- Mobile
- Lazy Load
- JavaScript
- Renderizado
- Caché
- Compresión
- CDN

---

# 10. Autoridad

Evaluar:

- Backlinks
- Dominios referidos
- Anchor Text
- Autoridad
- Enlaces tóxicos

Herramientas:

- Ahrefs
- SEMrush

---

# 11. Experiencia de usuario

Analizar:

- Navegación.
- Búsqueda.
- Tiempo hasta encontrar un producto.
- Legibilidad.
- Responsive.
- Accesibilidad.

SEO y UX deben analizarse conjuntamente.

---

# 12. Conversión

Preguntas:

¿La página convierte?

¿Existe CTA claro?

¿Genera confianza?

¿Facilita el contacto?

¿Reduce fricción?

---

# 13. Google Business Profile

Analizar:

- Reseñas.
- Fotografías.
- Publicaciones.
- Productos.
- Servicios.
- Preguntas.
- Categorías.

---

# 14. Contenido estratégico

Evaluar:

- Blog.
- Guías.
- Comparativas.
- Casos reales.
- FAQ.
- Vídeo.
- Recursos descargables.

---

# 15. Competencia

Comparar siempre con:

- Competidor principal.
- Competidor nacional.
- Referencia internacional.

Nunca analizar una web de forma aislada.

---

# 16. Priorización

Clasificar todas las mejoras.

## Alta prioridad

Mayor impacto.

Menor esfuerzo.

---

## Media prioridad

Impacto medio.

---

## Baja prioridad

Mejoras futuras.

---

# 17. Resultado esperado

Toda auditoría deberá finalizar con:

- Resumen ejecutivo.
- Problemas detectados.
- Oportunidades.
- Plan de acción.
- Priorización.
- Quick Wins.
- Roadmap.

Nunca terminar únicamente con una lista de errores.

El objetivo del SEO es aumentar negocio, no únicamente tráfico.
