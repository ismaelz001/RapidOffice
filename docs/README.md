# Ofiponiente Digital — Documentación

Antes de modificar código, diseño, SEO, ecommerce, configurador o automatizaciones, se debe revisar la documentación relevante.

## Estructura

- `master/`: visión principal del proyecto.
- `business/`: modelo de negocio y análisis de mercado.
- `frameworks/`: criterios estratégicos por área.
- `standards/`: reglas obligatorias para piezas concretas.
- `playbooks/`: procesos de ejecución.
- `research/`: auditorías, competencia y análisis.
- `knowledge/`: conocimiento histórico o externo relevante.
- `decisions/`: decisiones cerradas del proyecto.
- `templates/`: plantillas reutilizables.
- `archive/`: material histórico o previews no activos.

---

## Regla principal

Ofiponiente Digital no es solo una web ni una tienda online.

Es un canal digital para captar oportunidades comerciales, vender mejor y construir una marca fuerte en mobiliario profesional de oficina.
